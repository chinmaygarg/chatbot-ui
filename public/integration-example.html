Ready-to-use examples for integrating the chatbot widget into your website

üöÄ Quick Integration (2 steps)

Step 1: Add the widget container and script to your HTML:

<!-- Add this where you want the chatbot to appear -->
<div id="chatbot-widget"></div>

<!-- Load the widget script -->
<script src="https://your-cdn.com/chatbot-widget.umd.js"></script>

Step 2: Initialize with your API configuration:

<script>
  ChatbotWidget.init({
    apiEndpoint: 'https://your-api.com/chat',
    authToken: 'your-api-token',
    theme: 'default',
    welcomeMessage: 'Hello! How can I help you today?'
  });
</script>

  customHeaders: {
    'X-API-Version': '1.0',
    'X-Client-ID': 'web-widget'
  }
});

‚öôÔ∏è Configuration Options

Required Options

  • apiEndpoint - Your chat API endpoint URL

Authentication (choose one)

  • authToken - Bearer token for Authorization header
  • apiKey - API key for X-API-Key header

Appearance Options

  • theme - Theme: 'default', 'dark', 'ocean', 'sunset'
  • primaryColor - Custom primary color (hex)
  • position - Widget position: 'bottom-right', 'bottom-left'

Behavior Options

  • welcomeMessage - Custom welcome message
  • autoOpen - Auto-open chat on page load (boolean)
  • enableVoiceInput - Enable voice input (boolean, default: true)
  • streamingEnabled - Enable streaming responses (boolean, default: true)
  • maxMessageLength - Max characters per message (number, default: 1000)

Advanced Options

  • debug - Enable debug logging (boolean)
  • timeout - Request timeout in milliseconds (default: 30000)
  • retryAttempts - Number of retry attempts (default: 3)
  • customHeaders - Additional HTTP headers (object)

üéÆ Widget Control Methods

üì° Available Methods

// Initialize the widget
ChatbotWidget.init(config);

// Control widget visibility
ChatbotWidget.open();
ChatbotWidget.close();

// Send messages programmatically
ChatbotWidget.sendMessage('Hello from external code!');

// Update configuration
ChatbotWidget.updateConfig({ theme: 'dark' });

// Destroy widget
ChatbotWidget.destroy();

// Listen to events
window.addEventListener('chatbot:opened', () => console.log('Chat opened'));
window.addEventListener('chatbot:closed', () => console.log('Chat closed'));
window.addEventListener('chatbot:message', (e) => console.log('Message:', e.detail));

üîí Security Best Practices

  • Always use HTTPS in production
  • Store API tokens securely (never in client-side code)
  • Implement rate limiting on your backend
  • Use environment variables for sensitive configuration
  • Configure Content Security Policy (CSP) headers
  • Validate and sanitize all user inputs
